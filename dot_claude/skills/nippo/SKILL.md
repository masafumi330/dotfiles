# nippo — 日報管理スキル

ユーザーの入力: `$ARGUMENTS`

- **第1引数**（サブコマンド）: `init` / `check` / `add` / `memo` / `show` / `summary`
- **第2引数以降**（オプション）: タスク名やメモ本文など

第1引数を見てどのサブコマンドかを判断し、対応する処理を実行する。  
引数がない場合や不明な場合は使い方を表示する。

---

## 共通設定

- 日報ディレクトリ: `~/nippo/`
- ファイル名: `YYYY-MM-DD.md`（今日の日付は `date +%Y-%m-%d` で取得）
- 今日のファイルパス: `~/nippo/$(date +%Y-%m-%d).md`
- ディレクトリが存在しない場合は自動作成: `mkdir -p ~/nippo`

## テンプレート

See [template.md](template.md)

## `init` — 朝：今日の日報を作成

1. 今日のファイルが既に存在する場合は内容を表示して終了
2. `~/nippo/*.md` をソートして最新ファイルを特定（月曜・連休明けを考慮）
3. 最新ファイルの `# 明日やること` セクションを読み込み、引き継ぎタスクとして取得
4. 新規ファイルをテンプレートで作成し、引き継ぎタスクを `- [ ] <タスク名> [--:--]` 形式で `# 今日やること` の `[午前]` ブロックに書き込む
5. 第2引数以降にタスク名が与えられていればそれも `[午前]` に追記する（`- [ ] <タスク名> [--:--]`）
6. 追加タスクをインタラクティブに確認する:
   - 「他に今日やることはありますか？（なければそのままEnter）」
   - 入力があれば「午前・午後どちらに追加しますか？（am/pm）」と聞いてから該当ブロックに追記し、「他にありますか？」を繰り返す
7. 作成した日報を全文表示

**前日ファイルが存在しない場合（初回など）**: 引き継ぎなしで新規作成する

---

## `check` — タスク完了をチェック

1. 今日のファイルを読み込む（なければ「先に `/nippo init` を実行してください」）
2. 未完了タスク（`- [ ]`）を番号付きで一覧表示。`[午前]` / `[午後]` のどちらに属するかも示す
3. 第2引数でタスク番号が与えられていればそれを使用。なければインタラクティブに選ばせる
4. 現在時刻を取得: `date +%H:%M`
5. 該当行を更新:
   - `- [ ] <タスク名> [--:--]` → `- [x] <タスク名> [HH:MM]`
6. 「備考・メモはありますか？（なければEnter）」と聞き、入力があれば `# 学び・気づき・悩み・メモ` に追記
7. 更新後の `# 今日やること` セクションを表示

---

## `add` — 割り込みタスクを追加

引数を解析し、タスク名と追加先ブロックを決定してから追記する。

### 引数の解析

**ブロック指定（am/pm）**:
- 引数の末尾が単独の `am` または `pm` であれば、全タスクをそのブロックに追加する
- ブロック指定がなければ「午前・午後どちらに追加しますか？（am/pm）」とインタラクティブに確認する

**タスクの解析**:
- `- タスク名` 形式の行が複数あればリストとして複数タスクを解析する
- それ以外は引数全体（ブロック指定の `am`/`pm` を除く）をひとつのタスク名として扱う
- タスク名がなければインタラクティブに確認する

**例**:
- `add 朝会 am` → 「朝会」を午前に追加
- `add 朝会` → タスク名「朝会」、ブロックをインタラクティブに確認
- `add\n- タスクA\n- タスクB\npm` → タスクA・タスクBを両方午後に追加
- `add\n- タスクA\n- タスクB` → タスクA・タスクBを解析、ブロックをインタラクティブに確認

### 追記フォーマット

決定したブロック（`[午前]` または `[午後]`）の末尾に追記する（複数タスクは1行ずつ）:

```
- [ ] 差し込み: <タスク名> [--:--]
```

### 完了後

更新後の `# 今日やること` セクションを表示する。

## `memo` — 学び・気づきをメモ

1. 今日のファイルを読み込む
2. メモ内容を決定:
   - 第2引数以降が与えられていればそれをメモとして使用（スペース結合）
   - なければ「メモ内容を入力してください」とインタラクティブに聞く
3. `# 学び・気づき・悩み・メモ` セクションの末尾に追記:

   ```
   - <メモ内容>
   ```

4. 「メモを追加しました」と伝え、セクションを表示

---

## `show` — 現在の日報を表示

1. 今日のファイルを読み込む（なければ「先に `/nippo init` を実行してください」）
2. ファイルの全内容をそのまま表示

---

## `summary` — 終業処理

1. 今日のファイルを読み込む
2. `[午前]` / `[午後]` それぞれのブロック内で完了タスクを時系列順に並べ替える
3. 完了時刻から、タスクにかかった時間を計算し、(--h--m)形式で書く
4. 未完了タスク（`[午前]` / `[午後]` 両方）を「明日やること」に移動（ブロックラベルは付けない）
5. 「明日やることを追加しますか？（なければEnter）」と聞き、入力があれば追記
6. メモ欄の部分は、ログや構造はなるだけ変えず、誤字脱字の修正、インデントやフォーマットの調整、関連するメモは近くにまとめる、に留める。なるだけ綺麗にはしたいが、情報の欠落は抑えたいため。
7. 最終的な日報の全内容を表示して完了

## ファイル操作の原則

- 読み込み: Read ツールを使用
- 編集: Edit ツールで対象セクションの該当行のみ書き換える
- 時刻取得: `date +%H:%M`（Bashで実行）
- 最新ファイル検索: `ls ~/nippo/*.md 2>/dev/null | sort | tail -1`

## セクション操作

各セクションは `#` で始まる見出し行で区切られる。  
`# 今日やること` の中は `[午前]` / `[午後]` のラベル行でさらに2つのブロックに分かれる。  
編集時は見出し行・ラベル行を変更せず、各ブロック内のコンテンツのみを操作する。

`[午前]` ブロック = `[午前]` ラベル行の直後から `[午後]` ラベル行の直前まで  
`[午後]` ブロック = `[午後]` ラベル行の直後から次の `#` 見出し行の直前まで
