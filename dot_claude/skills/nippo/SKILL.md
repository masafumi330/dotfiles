# nippo — 日報管理スキル

ユーザーの入力: `$ARGUMENTS`

- **第1引数**（サブコマンド）: `init` / `check` / `add` / `memo` / `summary` / `asakai` / `weekly` / `monthly`
- **第2引数以降**（オプション）: タスク名やメモ本文など

第1引数を見てどのサブコマンドかを判断し、対応する処理を実行する。  
引数がない場合や不明な場合は使い方を表示する。

---

## 共通設定

- 日報ディレクトリ: `~/nippo/daily/`
- ファイル名: `YYYY-MM-DD.md`（今日の日付は `date +%Y-%m-%d` で取得）
- 今日のファイルパス: `~/nippo/daily/$(date +%Y-%m-%d).md`
- ディレクトリが存在しない場合は自動作成: `mkdir -p ~/nippo/daily`

## テンプレート

See [template.md](template.md)

## `init` — 朝：今日の日報を作成

1. 今日のファイルが既に存在する場合は内容を表示して終了
2. `~/nippo/daily/*.md` をソートして最新ファイルを特定（月曜・連休明けを考慮）
3. 最新ファイルの `# 明日やること` セクションを読み込み、引き継ぎタスクとして取得
4. 新規ファイルをテンプレートで作成し、引き継ぎタスクを `- [ ] <タスク名> [--:--]` 形式で `# 今日やること` の `[午前]` ブロックに書き込む
5. 第2引数以降にタスク名が与えられていればそれも `[午前]` に追記する（`- [ ] <タスク名> [--:--]`）
6. 追加タスクをインタラクティブに確認する:
   - 「他に今日やることはありますか？（なければそのままEnter）」
   - 入力があれば `[午前]` に追記し、「他にありますか？」を繰り返す
7. 作成した日報を全文表示

**前日ファイルが存在しない場合（初回など）**: 引き継ぎなしで新規作成する

---

## `check` — タスク完了をチェック

1. 今日のファイルを読み込む（なければ「先に `/nippo init` を実行してください」）
2. 未完了タスク（`- [ ]`）を番号付きで一覧表示。`[午前]` / `[午後]` のどちらに属するかも示す
3. 第2引数でタスク番号が与えられていればそれを使用。なければインタラクティブに選ばせる
4. 現在時刻を取得: `date +%H:%M`
5. 該当行を更新:
   - `- [ ] <タスク名> [--:--]` → `- [x] <タスク名> [HH:MM]`
6. 「備考・メモはありますか？（なければEnter）」と聞き、入力があれば `# 学び・気づき・悩み・メモ` に追記
7. 更新後の `# 今日やること` セクションを表示

---

## `add` — 割り込みタスクを追加

1. 今日のファイルを読み込む（なければ「先に `/nippo init` を実行してください」）
2. タスク名を決定:
   - 第2引数以降が与えられていればそれをタスク名として使用
   - `- タスク名` 形式の行が複数あればリストとして複数タスクを解析する
   - タスク名がなければインタラクティブに確認する
3. `[午前]` ブロックの末尾に追記する（複数タスクは1行ずつ）:
   ```
   - [ ] <タスク名> [--:--]
   ```
4. 更新後の `# 今日やること` セクションを表示する

---

## `memo` — 学び・気づきをメモ

1. 今日のファイルを読み込む
2. メモ内容を決定:
   - 第2引数以降が与えられていればそれをメモとして使用（スペース結合）
   - なければ「メモ内容を入力してください」とインタラクティブに聞く
3. `# 学び・気づき・悩み・メモ` セクションの末尾に追記:

   ```
   - <メモ内容>
   ```

4. 「メモを追加しました」と伝え、セクションを表示

---

## `asakai` — 朝会報告文を生成

1. 今日のファイルと、直近の前日ファイルを読み込む
2. 以下の形式で報告文を生成して表示する:

```
【昨日やったこと】
・<前日の完了タスク一覧>

【今日やること】
・<今日の完了済みタスク>（完了）
・<今日の未完了タスク>

【課題・共有事項】
・<前日メモの【課題】や相談事項があれば抜粋。なければ省略>
```

- 前日ファイルが存在しない場合は「昨日やったこと」を省略
- メモ欄に `【課題】` `【相談】` などのキーワードがあれば「課題・共有事項」に含める。なければセクションごと省略

---

## `summary` — 終業処理

1. 今日のファイルを読み込む
2. `[午前]` / `[午後]` それぞれのブロック内で完了タスクを時系列順に並べ替える
3. 完了時刻から、タスクにかかった時間を計算し、(--h--m)形式で書く
4. 未完了タスク（`[午前]` / `[午後]` 両方）を「明日やること」に移動（ブロックラベルは付けない）
5. 「明日やることを追加しますか？（なければEnter）」と聞き、入力があれば追記
6. メモ欄の部分は、ログや構造はなるだけ変えず、誤字脱字の修正、インデントやフォーマットの調整、関連するメモは近くにまとめる、に留める。なるだけ綺麗にはしたいが、情報の欠落は抑えたいため。
7. 最終的な日報の全内容を表示して完了

## `weekly` — 週次振り返りを作成

対象週の日報をまとめて振り返りファイルを生成する。

### 対象期間の決定

1週間のサイクルは **木曜〜水曜**（木曜に朝会で振り返りをするため）。

- 引数なし: 直前の木曜〜水曜（先週）を対象とする
- 引数あり: `YYYY-MM-DD` 形式で任意の日付を指定すると、その日を含む木〜水を対象とする

### 処理手順

1. 対象期間（木〜水）の `~/nippo/daily/YYYY-MM-DD.md` を全て読み込む
2. 存在しない日付のファイルはスキップ（週末・祝日等）
3. [weekly_template.md](weekly_template.md) を使って `~/nippo/weekly/<YYYYMMDD>-<YYYYMMDD>.md` を生成する

ファイルパスの日付部分: 開始日（木曜）と終了日（水曜）を `YYYYMMDD` 形式（ハイフンなし）で結合。例: `20250522-20250528.md`

- 完了タスクは日付順・時系列順に並べる
- メモは重複・類似内容をまとめて読みやすく整理する（情報の欠落は最小限に）
- KPT セクションを生成する:
  - **Keep**: 今週うまくいったこと・継続したい行動や習慣（完了タスクの傾向・メモの良かった点から抽出）
  - **Problem**: うまくいかなかったこと・改善したい点（未完了タスク・メモ内の課題・悩みから抽出）
  - **Try**: Problemへの具体的な対策や、来週試したいこと
- 「来週に向けて」は KPT の Try を踏まえた内容にする
- 既にファイルが存在する場合は内容を表示して上書き確認をとる

---

## `monthly` — 月次振り返りを作成

対象月の週次振り返りファイル（なければ日報ファイル）をまとめて月次レポートを生成する。

### 対象月の決定

- 引数なし: 前月を対象とする（例: 実行日が6月なら5月分）
- 引数あり: `YYYY-MM` または `YYYYMM` 形式で指定（例: `monthly 2025-05`）

### 処理手順

1. `~/nippo/weekly/` に対象月を含む週次ファイルがあれば全て読み込む
2. 週次ファイルが存在しない場合は `~/nippo/daily/YYYY-MM-*.md` を直接読み込む
3. [monthly_template.md](monthly_template.md) を使って `~/nippo/monthly/<YYYYMM>.md` を生成する

- 週ごとにセクションを分けて完了タスクを整理する
- KPT セクションを生成する:
  - **Keep**: 今月を通じてうまくいったこと・継続したい行動や習慣（各週の Keep や成果から統合）
  - **Problem**: 月を通じた課題・未解決事項・繰り返し出てきた問題（各週の Problem や悩みから統合）
  - **Try**: Problemへの対策や来月試したいこと（各週の Try を踏まえて具体化）
- 「来月に向けて」は KPT の Try を踏まえた内容にする
- 既にファイルが存在する場合は内容を表示して上書き確認をとる

## ファイル操作の原則

- 読み込み: Read ツールを使用
- 編集: Edit ツールで対象セクションの該当行のみ書き換える
- 時刻取得: `date +%H:%M`（Bashで実行）
- 最新ファイル検索: `ls ~/nippo/daily/*.md 2>/dev/null | sort | tail -1`

## セクション操作

各セクションは `#` で始まる見出し行で区切られる。  
`# 今日やること` の中は `[午前]` / `[午後]` のラベル行でさらに2つのブロックに分かれる。  
編集時は見出し行・ラベル行を変更せず、各ブロック内のコンテンツのみを操作する。

`[午前]` ブロック = `[午前]` ラベル行の直後から `[午後]` ラベル行の直前まで  
`[午後]` ブロック = `[午後]` ラベル行の直後から次の `#` 見出し行の直前まで
